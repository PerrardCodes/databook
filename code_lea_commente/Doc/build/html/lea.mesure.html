bulle(filename, fx=0.0, fps=1.0, D=0, xmin=0, xmax=-1, ymin=0, ymax=-1, x0=0, y0=0, p_im=0, nb_im=0)¶

La fonction bulle est tirée du code de fluids2d.backlight de danjruth Elle ajoute dynamiquement dans un fichier hdf5 passé en paramètre

Paramètres:
  • filename (h5py files) – Fichier où seront stocké les données calculées
  • xmin (int) – Pour crop l’image, de base à 0
  • xmax (int) – Pour crop l’image, de base à -1 et si non définit il prend le maximum de l’axe x
  • ymin (int) – Pour crop l’image, de base à 0
  • ymax (int) – Pour crop l’image, de base à -1 et si non définit il prend le maximum de l’axe y
  • x0 (int) – Le centre en x, il faut indiquer le centre avant que l’image ne soit crop.
  • y0 (int) – Le centre en y, il faut indiquer le centre avant que l’image ne soit crop.
  • p_im (int) – Première image à traiter
  • nb_im (int) – Dernière image à traiter, si égale à 0 elle est mise au nombre maximum d’image

Pour les autres paramètres il récupère ceux nécessaire dans le fichier data, sinon il met les valeurs passées en paramètre

Ajoute un paramètre à Contour, si c’est un objet il ajoute l’attribut avec pour nom le nom de la classe qu’on a récupéré grâce à la méthode get_name() sinon il l’ajoute avec le nom donné en paramètre

contour_RAM(fx=0.0, fps=1.0, D=0, xmin=0, xmax=-1, ymin=0, ymax=-1, x0=0, y0=0, im_save=50, adresse_s='', rmin=10, rmax=30, B=5, thetamin=0.7, thetamax=0.15, nb_im=0)¶

Cette fonction récupère les contours d’un ballon et grâce à la fonction :func: contour. Lors de son utilisation avec un film celui-ci est stocké entièrement dans la RAM (c’est à dire qu’il charge en premier toutes les images ou tout le film puis le traite en suite), il faut préférer utiliser contour_instant() ou contour_multi_proc() pour l’utilisation du multiprocessing python

Paramètres:
  • fx (float) – spacial scale
  • fps (float) – Utilisé pour calculer le ft
  • D (float) – Diametre initial
  • xmin (int) – Pour crop l’image, de base à 0
  • xmax (int) – Pour crop l’image, de base à -1 et si non définit il prend le maximum de l’axe x
  • ymin (int) – Pour crop l’image, de base à 0
  • ymax (int) – Pour crop l’image, de base à -1 et si non définit il prend le maximum de l’axe y
  • x0 (int) – Le centre en x, il faut indiquer le centre avant que l’image ne soit crop.
  • y0 (int) – Le centre en y, il faut indiquer le centre avant que l’image ne soit crop.
  • im_save (int) – Image à sauvegarder
  • adresse_s (adresse du fichier) – Où sauvegarder l’image
  • rmin (int) – rmin et rmax servent à savoir à quelle distance on doit mettre les points
  • rmax (int) – rmin et rmax servent à savoir à quelle distance on doit mettre les points
  • N (int) – N est le nombre de droite (N = pi* (rmin-rmax) *B)
  • nr (int) – nr est le nombre de point par droite (nr = (rmax-rmin)*B)
  • nb_im (int) – Dernière image à traiter, si égale à 0 elle est mise au nombre maximum d’image
