lea.hdf5.h5py_convert.h5py_in_Data(f)¶

Prend en paramètre un fichier HDF5 et renvoie le fichier Data qui est dedans

Paramètres:f (Files h5py) – Fichier ouvert
Renvoie:Data Object

Avertissement

Ne marche pas récursivement ! Le sous-dossier Data doit être à la racine.

Note

Si le fichier HDF5 contient un objet Mesure il faut utiliser h5py_in_Mesure()

lea.hdf5.h5py_convert.h5py_in_Mesure(f)¶
Paramètres:f (Files h5py) – Fichier ouvert
Renvoie:Mesure Object
  • Si jamais le fichier a un sous dossier Contour il va appeler h5py_in_Contour()
  • Si jamais le fichier a un sous dossier Bulles il va appeler h5py_in_Bulles()
  • Si jamais le fichier a un sous dossier PIV3D il va appeler h5py_in_piv3d()
  • Si jamais le fichier a un sous dossier Volume il va appeler h5py_in_Volume()

lea.hdf5.h5py_convert.obj_in_h5py(object, file, group=None, point='', attr='')¶
Paramètres:
  • object (Python Object) – Objet à transférer en HDF5
  • file – fichier dans lequel il faut sauvegarder le fichier hdf5
Renvoie:

None

Ecrit l’objet dans le fichier au furt et à mesure qu’il descent récursivement dans l’objets passé en paramètre.

Necessite l’utilisation de ouverture_fichier() ou de file_name_in_dir()

Note

Les paramètres group, point, attr sont utilisés pour les récursions.
